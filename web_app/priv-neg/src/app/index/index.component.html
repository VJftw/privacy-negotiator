Privacy Manager

A way to manage collective privacy on Facebook.

(Logs in with Facebook)

More information

Why?

When we upload photos to Facebook, tag our friends and family; only we, the uploader, can modify who can see those photos. This gives zero power to the people depicted in those photos to autonomously manage their privacy and in many cases their privacy is breached. No social network currently offers a way to collectively manage privacy between co-owners of media.

What this tool does

This web application utilises the Facebook Graph API to:

  • Present photos that you are tagged in:
    • Greyscaled photos are publicly viewable and searchable by anyone. The uploader's of those need to authorize this tool for us to mediate on them.
    • Coloured photos are considered negotiable. With these, you can:
      • View the users that are tagged.
      • Modify the categories/context of the photo. This is shared between all users.
      • View who is allowed to see the photo.
      • View who is blocked from seeing the photo.
      • View which co-owners are in conflict and whom they are in conflict over.
  • Present your friends (that have authorised this tool) in automatically generated cliques.
    • Cliques are determined by the Clique Percolation Method (CPM), and are stored in bi-partites (A sequential algorithm for fast clique percolation).
    • You can manage your preferences for what contexts/categories a clique is allowed to view.

What this tool can't do

This tool is limited by any limitations that the Facebook Graph API exhibits.

  • Present existing privacy settings for a photo. The Graph API does not allow privacy to be read from a photo. 1
  • Show all friends. The Graph API only allows the tool to see friends that have mutually authorised it. 2
  • Determine categories/context for a photo automatically. This would be a whole other project as there currently isn't any AI that can interpret photos well. They can describe objects but cannot decide on suitable contexts.

There may be some bugs as I've only had 1 month to create this, resulting in no test coverage. This should improve as I commit further patches.
